Capítulo 8

Tratamento de Erros

Programas reais lidam com entradas erradas, arquivos ausentes, divisões por zero e respostas inesperadas. Tratar erros é impedir que uma falha previsível derrube toda a experiência.

Ideia-chave: erro não é vergonha; erro sem tratamento é problema. Um bom programa orienta o usuário e continua seguro sempre que possível.

try, except e finally

Use try para envolver uma operação que pode falhar. Use except para responder a falhas específicas. Use finally quando algo deve acontecer com erro ou sem erro.

try: numero = int(input("Digite um número: ")) resultado = 10 / numero print(f"10 / {numero} = {resultado}") except ValueError: print("Erro: você não digitou um número válido.") except ZeroDivisionError: print("Erro: divisão por zero não é permitida.") finally: print("Fim do programa.")
Evite except genérico

Escrever apenas except: pode esconder erros importantes. Prefira tratar exceções específicas, como ValueError e ZeroDivisionError.

Validar antes também ajuda

Nem tudo precisa virar exceção. Quando uma regra é simples, você pode validar com if antes de executar a operação.

numero = int(input("Digite um número: ")) if numero == 0: print("Não é possível dividir por zero.") else: print(10 / numero)

Função segura

def ler_inteiro(mensagem): while True: try: return int(input(mensagem)) except ValueError: print("Digite apenas números inteiros.") idade = ler_inteiro("Idade: ") print(f"Idade informada: {idade}")
Exercício 3.8 — Entrada resistente

Adapte a função ler_inteiro para só aceitar valores entre 0 e 120.
