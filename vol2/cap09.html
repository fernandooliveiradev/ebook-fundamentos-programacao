Capítulo 9

Tratamento de Dados

Programas quase nunca recebem dados perfeitos. O usuário digita texto, números chegam como strings, campos vêm vazios e nomes aparecem com espaços extras. Tratar dados é preparar a entrada antes de confiar nela.

A lógica de um sistema pode estar correta e ainda assim falhar se os dados estiverem no tipo errado. Antes de calcular, comparar ou salvar, valide e converta.

O dado bruto passa por conversão e validação antes de entrar na regra de negócio.

Conversões entre tipos

Conversão é transformar um valor de um tipo para outro. O caso mais comum é receber texto e converter para número antes de calcular.

textoNumero = "42" numeroInteiro = CONVERTER_PARA_INTEIRO(textoNumero) numeroReal = CONVERTER_PARA_REAL(numeroInteiro)
Erro comum

Texto que parece número ainda é texto. Em muitas linguagens, "10" + "5" pode virar "105", não 15. Converta antes de operar.

Manipulação de strings

Strings são sequências de caracteres. Tratar texto envolve medir, cortar, procurar, padronizar maiúsculas/minúsculas e remover espaços desnecessários.

nome = "João Silva" ESCREVA "Tamanho: ", COMPRIMENTO(nome) ESCREVA "Primeiro nome: ", SUBSTRING(nome, 0, 4) ESCREVA "Maiúsculo: ", PARA_MAIUSCULO(nome) SE CONTEM(nome, "Silva") ENTÃO ESCREVA "Sobrenome encontrado" FIM SE

Validação

Validar é verificar se o dado serve para a regra. Um e-mail precisa ter formato aceitável, uma idade não pode ser negativa e uma nota escolar deve respeitar a escala combinada.

SE idade < 0 OU idade > 120 ENTÃO ESCREVA "Idade inválida" SENÃO ESCREVA "Idade aceita" FIM SE
Exercício 2.7 — Entrada confiável

Escreva um pseudocódigo que leia uma nota como texto, converta para real e valide se ela está entre 0 e 10.

Ver solução
LEIA notaTexto nota = CONVERTER_PARA_REAL(notaTexto) SE nota >= 0 E nota <= 10 ENTÃO ESCREVA "Nota válida" SENÃO ESCREVA "Nota inválida" FIM SE
