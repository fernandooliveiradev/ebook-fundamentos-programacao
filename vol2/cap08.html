Capítulo 8

Funções e Procedimentos

Quando um algoritmo cresce, repetir blocos parecidos deixa tudo difícil de ler. Funções (functions) e procedimentos (procedures) resolvem isso: eles dão nome a uma parte da solução e permitem reutilizá-la.

Conexão com abstração: no Volume 1, você viu que abstrair é esconder complexidade atrás de uma ideia simples. Uma função faz exatamente isso: por fora, você chama pelo nome; por dentro, ela executa os passos.

Procedimentos fazem; funções fazem e retornam um valor.

Procedimentos: quando a ação é o objetivo

Use procedimento quando você quer executar uma tarefa, como mostrar uma mensagem, limpar uma tela, imprimir um relatório ou validar uma etapa sem precisar guardar um resultado.

PROCEDIMENTO ExibirCabecalho(titulo: TEXTO) INICIO ESCREVA "========================" ESCREVA " ", titulo ESCREVA "========================" FIM PROCEDIMENTO ExibirCabecalho("Sistema de Vendas")

Funções: quando você precisa de uma resposta

Use função quando o bloco precisa retornar (return) um valor para outro ponto do algoritmo. O retorno permite compor soluções maiores.

FUNCAO CalcularMedia(n1, n2, n3: REAL): REAL VAR media: REAL INICIO media = (n1 + n2 + n3) / 3 RETORNE media FIM FUNCAO mediaFinal = CalcularMedia(7.5, 8.0, 6.5) ESCREVA "Média: ", mediaFinal
Regra prática

Se você consegue completar a frase “quero calcular...” ou “quero obter...”, provavelmente precisa de uma função. Se a frase é “quero exibir...”, “quero salvar...” ou “quero executar...”, um procedimento pode bastar.

Recursividade

Recursividade (recursion) acontece quando uma função chama a si mesma. Ela precisa de duas partes: um caso base (base case), que para a repetição, e um passo recursivo, que aproxima o problema desse caso base.

FUNCAO Fatorial(n: INTEIRO): INTEIRO INICIO SE n <= 1 ENTÃO RETORNE 1 SENÃO RETORNE n * Fatorial(n - 1) FIM SE FIM FUNCAO // Fatorial(5) = 5 × 4 × 3 × 2 × 1 = 120
Exercício 2.6 — Modularização

Crie uma função chamada EhPar que receba um número inteiro e retorne VERDADEIRO se ele for par.

Ver solução
FUNCAO EhPar(numero: INTEIRO): LOGICO INICIO RETORNE numero % 2 == 0 FIM FUNCAO

Funções organizam o comportamento do algoritmo. Para que funcionem bem, os dados que entram e saem delas também precisam estar corretos: tipos coerentes, textos tratados e validações no ponto certo.
