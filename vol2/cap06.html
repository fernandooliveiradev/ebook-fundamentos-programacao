Capítulo 6

Laços de Repetição

Um laço de repetição, em inglês loop, serve para executar a mesma ideia várias vezes sem copiar o mesmo bloco de código. Em vez de escrever dez comandos parecidos, você escreve uma regra de repetição.

Pense em uma chamada escolar. O professor não escreve uma instrução diferente para cada aluno. Ele segue uma regra: para cada nome da lista, chamar o aluno e marcar presença. Esse “para cada” é exatamente o tipo de raciocínio que um laço representa.

A ideia antes do código

Todo laço tem três perguntas importantes. Se o aluno consegue responder essas perguntas, fica muito mais fácil escolher entre PARA, ENQUANTO e FAÇA-ENQUANTO.

O que repete?

Identifique o bloco de código (code block) que será executado mais de uma vez.

Quando começa?

Defina o valor inicial ou a primeira situação da repetição.

Quando para?

Determine a condição (condition) que encerra o laço.

Leitura simples

Um laço pode ser lido assim: “enquanto esta regra for verdadeira, repita este bloco” ou “para cada valor dentro deste intervalo, execute este bloco”.

PARA (for): quando sabemos quantas vezes repetir

Use PARA, em inglês for, quando a quantidade de repetições já está clara. Ele é comum para contar, fazer tabuadas, somar intervalos e percorrer posições conhecidas.

PARA contador DE inicio ATE fim PASSO incremento // bloco repetido FIM PARA

Como ler o PARA

O contador começa em um valor, muda a cada volta e para quando chega ao limite definido.

  1. Comece o contador.
  2. Execute o bloco.
  3. Atualize o contador.
  4. Verifique se ainda deve repetir.

Exemplo: tabuada

Na tabuada, sabemos que queremos repetir de 1 até 10. Por isso, PARA é a escolha mais direta.

ALGORITMO Tabuada VAR numero, i: INTEIRO INICIO ESCREVA "Digite um número:" LEIA numero PARA i DE 1 ATE 10 PASSO 1 ESCREVA numero, " x ", i, " = ", numero * i FIM PARA FIM

ENQUANTO (while): quando a repetição depende de uma condição

Use ENQUANTO, em inglês while, quando não sabemos de antemão quantas vezes o bloco vai repetir. O laço continua enquanto a condição for verdadeira.

Ponto importante

No ENQUANTO, a condição é testada antes da primeira execução. Se ela já começar falsa, o bloco não executa nenhuma vez.

Exemplo: validar senha

Neste exemplo, não sabemos se o usuário vai acertar na primeira, segunda ou terceira tentativa. Por isso usamos uma condição: repetir enquanto a senha estiver errada e ainda houver tentativas.

ALGORITMO ValidaSenha VAR senhaCorreta, tentativa: TEXTO VAR tentativas: INTEIRO INICIO senhaCorreta = "1234" tentativa = "" tentativas = 0 ENQUANTO tentativa <> senhaCorreta E tentativas < 3 FACA ESCREVA "Digite a senha:" LEIA tentativa tentativas = tentativas + 1 FIM ENQUANTO SE tentativa = senhaCorreta ENTAO ESCREVA "Acesso liberado" SENAO ESCREVA "Acesso bloqueado" FIM SE FIM
Atenção: loop infinito

Loop infinito (infinite loop) acontece quando a condição nunca fica falsa. Sempre garanta que algo dentro do laço altere a variável que controla a repetição.

FAÇA-ENQUANTO (do while): quando precisa executar pelo menos uma vez

Use FAÇA-ENQUANTO, em inglês do while, quando o bloco precisa rodar antes do teste. Ele é comum em menus, porque o usuário precisa ver as opções pelo menos uma vez.

ALGORITMO Menu VAR opcao: INTEIRO INICIO FACA ESCREVA "1 - Cadastrar" ESCREVA "2 - Consultar" ESCREVA "3 - Sair" LEIA opcao ENQUANTO opcao <> 3 FIM

Erros comuns de iniciantes

Esquecer de atualizar

Se o contador ou a variável de controle não muda, o laço pode nunca terminar.

Escolher o laço errado

Se você sabe que vai de 1 a 10, use PARA. Se depende de uma resposta do usuário, geralmente use ENQUANTO.

Colocar código fora do bloco

Indentação e organização visual ajudam a mostrar o que repete e o que executa apenas uma vez.

Exercício 2.3 — Laços de repetição

Escreva um pseudocódigo para somar os números de 1 a 100 e mostrar o resultado final.

Ver solução
soma = 0 PARA i DE 1 ATE 100 PASSO 1 soma = soma + i FIM PARA ESCREVA "A soma é: ", soma

Resultado: 5050.
