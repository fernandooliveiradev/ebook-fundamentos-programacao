Capítulo 7

Estruturas de Dados

Estruturas de dados, em inglês data structures, são formas de organizar informações para que o algoritmo consiga guardar, buscar, alterar e percorrer dados com clareza.

Imagine uma escola tentando guardar notas. Se existe apenas uma nota, uma variável resolve. Mas se existem notas de uma turma inteira, precisamos de uma organização melhor: lista, vetor, matriz, pilha ou dicionário, dependendo do problema.

Por que não usar só variáveis?

Uma variável (variable) guarda um valor. Isso funciona bem para uma idade, uma média ou um preço. Mas programas reais quase sempre lidam com conjuntos: vários alunos, vários produtos, várias tentativas, várias tarefas.

Um valor

Use uma variável simples: idade, preco, media.

Vários valores em ordem

Use vetor, lista ou matriz. A posição do dado importa.

Valores com identificação

Use dicionário quando você quer buscar por nome, código ou chave.

Ideia central

Escolher uma estrutura de dados é escolher uma forma de fazer perguntas ao dado: “qual é o terceiro item?”, “qual telefone pertence a Maria?”, “qual foi o último item colocado?”.

Vetor (array): vários valores do mesmo tipo

Um vetor, em inglês array, guarda vários elementos em sequência. Cada elemento fica em uma posição chamada índice (index). No pseudocódigo deste e-book, usaremos índices começando em 1.

Como pensar em um vetor

Um vetor é como uma fileira de caixas numeradas. Cada caixa guarda um valor, e você acessa o valor pelo número da posição.

  1. Crie o vetor com um tamanho.
  2. Guarde valores em posições.
  3. Acesse uma posição quando precisar.
  4. Percorra o vetor com um laço.

Exemplo: média de cinco notas

ALGORITMO MediaNotas VAR notas: VETOR[1..5] DE REAL soma, media: REAL i: INTEIRO INICIO soma = 0 PARA i DE 1 ATE 5 ESCREVA "Digite a nota ", i, ":" LEIA notas[i] soma = soma + notas[i] FIM PARA media = soma / 5 ESCREVA "Média: ", media FIM

Matriz (two-dimensional array): linhas e colunas

Uma matriz, em inglês two-dimensional array, organiza dados em linhas e colunas. Ela é útil quando o dado tem formato de tabela, grade, planilha, mapa ou tabuleiro.

Exemplo: matriz 3x3

ALGORITMO MatrizExemplo VAR matriz: VETOR[1..3][1..3] DE INTEIRO linha, coluna: INTEIRO INICIO PARA linha DE 1 ATE 3 PARA coluna DE 1 ATE 3 ESCREVA "Digite o valor da posição ", linha, ",", coluna LEIA matriz[linha][coluna] FIM PARA FIM PARA FIM
Leitura prática

Vetor tem uma dimensão: posição. Matriz tem duas: linha e coluna. Por isso, normalmente usamos dois laços: um para as linhas e outro para as colunas.

Lista dinâmica (dynamic list): quando o tamanho pode mudar

Uma lista dinâmica, em inglês dynamic list, permite adicionar, inserir e remover itens durante a execução do algoritmo. Ela é útil quando não sabemos previamente quantos elementos existirão.

lista = [] lista.adicionar(10) lista.adicionar(20) lista.inserir(1, 15) // Resultado: [10, 15, 20] lista.remover(20) // Resultado: [10, 15]

Use lista quando os dados entram e saem com frequência: lista de tarefas, carrinho de compras, nomes inscritos em um evento ou itens de um pedido.

Pilha (stack): o último entra, o primeiro sai

Uma pilha, em inglês stack, segue a regra LIFO: Last In, First Out, ou seja, “o último a entrar é o primeiro a sair”.

Analogia simples

Pense em uma pilha de pratos. Você coloca um prato por cima e também retira primeiro o prato que está por cima.

pilha = [] pilha.empilhar("A") pilha.empilhar("B") pilha.empilhar("C") // Estrutura: [A, B, C] topo = pilha.desempilhar() // Retorna "C"

Dicionário (dictionary / map): buscar por chave

Um dicionário, em inglês dictionary ou map, guarda pares chave-valor (key-value pairs). Em vez de buscar pela posição, você busca pela chave.

agenda["João"] = "9999-1111" agenda["Maria"] = "9999-2222" telefoneJoao = agenda["João"] PARA cada nome EM agenda ESCREVA nome, ":", agenda[nome] FIM PARA

Dicionários são bons para cadastro, agenda, estoque, configuração de sistema e qualquer situação em que um identificador aponta para um valor.

Erros comuns de iniciantes

Usar variável demais

Se aparecem nota1, nota2, nota3, talvez um vetor resolva melhor.

Confundir índice com valor

O índice é a posição. O valor é o conteúdo guardado naquela posição.

Usar matriz sem necessidade

Se não existe linha e coluna, uma lista ou vetor costuma ser mais simples.

Exercício 2.4 — Escolhendo estruturas

Para cada situação, diga qual estrutura você usaria: vetor, matriz, lista dinâmica, pilha ou dicionário.

  • a) Guardar as notas de cinco avaliações.
  • b) Representar um tabuleiro 3x3.
  • c) Guardar telefones procurando pelo nome da pessoa.
  • d) Controlar uma lista de tarefas que pode receber novos itens.
  • e) Implementar a função “desfazer” de um editor.
Ver solução
  • a) Vetor — valores em sequência e quantidade conhecida.
  • b) Matriz — estrutura com linhas e colunas.
  • c) Dicionário — busca por chave, neste caso o nome.
  • d) Lista dinâmica — a quantidade de itens muda.
  • e) Pilha — a última ação feita deve ser a primeira a ser desfeita.

Com os dados organizados, também precisamos organizar comportamentos. Funções e procedimentos dão nome a blocos de lógica para que eles possam ser reaproveitados.
